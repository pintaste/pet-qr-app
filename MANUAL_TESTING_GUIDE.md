# Pet QR System - 手动测试指导

## 🎯 测试目标

本指导旨在帮助发现MVP版本的问题，包括bugs、使用场景问题、业务逻辑缺陷等，覆盖所有核心功能和边缘情况。

---

## 🔧 测试环境准备

### 前置条件
- Backend: http://localhost:8000 (运行中)
- Frontend: http://localhost:3003 (运行中)
- Database: PostgreSQL (运行中)
- 浏览器: Chrome/Safari (开启开发者工具)

### 测试工具
- **浏览器开发者工具**: 监控网络请求和控制台错误
- **手机扫码工具**: 测试QR码扫描
- **不同设备**: 手机、平板、桌面 (响应式测试)

---

## 📋 测试检查清单

### ✅ 每个测试项目评估标准
- **功能性**: 是否按预期工作
- **用户体验**: 操作是否直观
- **错误处理**: 异常情况处理是否合理
- **性能**: 响应速度是否可接受
- **安全性**: 数据保护是否充分

## 场景一: 用户认证流程

### 1.1 用户注册
**测试步骤:**
1. 访问 http://localhost:3003
2. 点击"注册"按钮
3. 填写注册信息
4. 提交注册表单

**测试用例:**
```
正常情况:
- 输入有效邮箱: test@example.com
- 输入强密码: Password123!
- 确认密码匹配
- 检查注册成功消息

边缘情况:
- 邮箱格式错误: invalid-email
- 密码太短: 123
- 密码不匹配
- 重复邮箱注册
- 网络断开情况
- 特殊字符输入: <script>alert('xss')</script>

预期结果:
✅ 成功注册并自动登录
✅ 错误信息清晰显示
✅ 表单验证正确
❌ 发现问题记录: [在此记录发现的bug]
```

### 1.2 用户登录
**测试步骤:**
1. 使用已注册账户登录
2. 验证JWT token获取
3. 检查自动跳转

**测试用例:**
```
正常情况:
- 正确邮箱密码登录
- 检查token存储
- 验证页面跳转

异常情况:
- 错误密码: wrongpassword
- 不存在的邮箱: nonexistent@test.com
- 空字段提交
- 网络超时情况
- SQL注入尝试: ' OR 1=1 --

预期结果:
✅ 登录成功获得访问权限
✅ 错误提示用户友好
✅ 安全防护有效
❌ 发现问题记录: [在此记录发现的bug]
```

### 1.3 Token刷新机制
**测试步骤:**
1. 登录后等待30分钟(或修改token过期时间测试)
2. 执行需要认证的操作
3. 观察token自动刷新

**重点检查:**
- 自动刷新是否正常

## 场景三: QR码生成与管理

### 3.1 QR码生成
**测试步骤:**
1. 选择已创建的宠物
2. 点击"生成QR码"
3. 设置PIN码
4. 生成并验证

**测试用例:**
```

边缘情况:
- PIN码格式: 非数字, 长度不对
- 重复为同一宠物生成QR码

预期结果:
✅ QR码成功生成
✅ PIN设置有效
✅ 代码唯一性保证
❌ 发现问题记录: [在此记录发现的bug]
```

**检查点:**

### 3.3 QR码管理
